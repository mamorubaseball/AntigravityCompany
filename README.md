📁 各事業部のREADME

以下は、各事業部内の細分化された部署とエージェントの役割、および連携プロトコルです。

1. 🎥 投資メディア事業

【ミッション】
日本株・米国株の専門チャンネルを運営し、広告収益・アフィリエイト・PR案件による収益を最大化する。

【所属部署とエージェント】

🇯🇵 日本株部署:

日本株アナリスト: 日経新聞、決算短信、適時開示などをスクレイピングし、注目銘柄や相場サマリーを作成する。

日本株チャンネル・ディレクター: アナリストのデータを元に、視聴維持率を意識した動画台本とサムネイル構成案を作成する。

🇺🇸 米国株部署:

米国株アナリスト: ブルームバーグ、SECファイリング（EDGAR）、米経済指標を分析し、トレンドをまとめる。

米国株チャンネル・ディレクター: アナリストのデータを元に、動画台本と構成案を作成する。

【連携フロー】

Input: 社長からの「今週の注目テーマ」や「取り上げるべきセクター」の指示。

Output: YouTube用動画台本・サムネイル素材一式、SNS告知用テキスト。

2. 📈 投資事業

【ミッション】
自己資本を用いた中期投資事業（売上目標：400万円）を達成し、安定したキャッシュフローを生み出す。

【所属部署とエージェント】

💹 スイングトレード部署:

マーケットリサーチャー: 日々のニュース、決算情報、アナリストレポートなどの一次情報を収集・分析し、スイングトレードの候補銘柄を抽出する。

スイングトレーダー: リサーチャーが収集した情報とテクニカル指標を元に売買シナリオを構築し、証券API経由で発注・決済を行う。

💰 資産管理部署:

ポートフォリオマネージャー: ポートフォリオ全体のボラティリティを監視し、損切りラインの徹底やポジションサイズの調整、資金管理ルールに基づく資産配分を提案する。

【連携フロー】

Input: 社長からの「リスク許容度」や「投資方針」の設定。投資メディア事業からの市場分析データの共有。

Output: トレード実行アラート、週次・月次の運用成績レポート。

3. 🤖 AI民主化事業

【ミッション】
AIの活用方法や教育コンテンツを発信し、チャンネルのグロースおよび将来的な商材（教材・プロンプト）販売、コミュニティ運営への導線を作る。

【所属部署とエージェント】

📣 メディア部署:

AIトレンドリサーチャー: 海外ニュース、論文、GitHub、Xのトレンドを監視し、最新AIツールの情報をリストアップする。

AI教育チャンネル・ディレクター: リサーチ結果を元に、一般向けに分かりやすいAIツールの解説台本や、販売用の「実践的プロンプト集」のドラフトを作成する。

SNS＆グロース担当: 動画の切り抜きや図解をX等に自動投稿し、集客を行う。

【連携フロー】

Input: 社長からの「現在注力すべきAI領域」の指示。

Output: AI解説動画の台本、教材ドラフト、日々のSNS発信。

4. 💻 アプリ開発事業部

【ミッション】
自社サービスの開発を最速で進め、MVPのローンチから継続的な機能改善（CI/CD）までを完全自律で実行する。

【所属部署とエージェント】

🛠️ 企画・UI/UX部署:

プロダクトマネージャー (PM): 社長のアイデアを要件定義書、タスクチケットに分解し、Agent Manager上でエンジニアエージェントに割り振る。

⚙️ エンジニアリング部署:

フロントエンド＆バックエンド・エンジニア: Antigravityエディタ内で非同期にコードを実装。相互にAPIの仕様をすり合わせながら自律的に開発を進める。

🧪 QA・DevOps部署:

QAテスト＆デプロイメント: 実装されたコードのバグを検知し、Browser AgentでUIを確認。問題がなければ本番環境へのデプロイを自動実行する。

【連携フロー】

Input: 社長からのアプリのアイデアや追加機能の要望。

Output: 動くアプリケーション（URL）、GitHubへのPull Request。

5. 👔 社長室

【ミッション】
4つの事業が並行して動く中、社長の脳内メモリをクリアに保ち、リスクを排除する。

【所属部署とエージェント】

AI専属秘書: 全部署のエージェントからの報告（トレード結果、各チャンネルの台本完成、アプリの進捗など）を1つのダッシュボードに集約。「社長が今決断すべきこと」だけを抽出して朝夕に報告する。

法務・コンプライアンスチェッカー: 投資YouTubeでの「投資助言業」に抵触する表現の修正提案や、PR案件におけるステマ規制（#PR表記）のチェック、アプリ利用規約の作成サポートを行う。

🔄 1日の標準的なワークフロー（社長の業務）

朝の意思決定 (Morning Approval):

AI秘書のダッシュボードを開き、投資事業が抽出した「本日の買い候補銘柄」を確認し、発注を承認。

投資メディア・AI民主化事業から上がってきた「本日の動画企画書」をチェックし、制作開始を指示。

アプリ開発のオーケストレーション (Dev Orchestration):

アプリ開発事業部のPMエージェントが作成したチケットを確認し、エンジニアリング部署に本日の実装タスクを走らせる。

コンテンツの最終確認とディープワーク (Review & Deep Work):

各メディア事業部から上がってきた台本・教材を最終チェック。

エージェントたちが自律稼働している間、人間である社長は「次の新しいサービス企画」や「事業ドメインの探求」という最高価値の業務に時間を使う。
